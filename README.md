### 🟢 Лаб. 1 — Вычислительная погрешность

> 📓 [`task_1.ipynb`](task_1.ipynb)

Вычисление суммы $S = \sum_{n=1}^{10^6} \frac{\pi}{n^2}$ двумя способами:

| Метод | Направление | Описание |
|:-----:|:-----------:|----------|
| ➡️ Прямое | $1 \to N$ | Сумма от малых индексов к большим |
| ⬅️ Обратное | $N \to 1$ | Сумма от больших индексов к малым |

📊 **Результат:** разница $\approx 1.66 \times 10^{-13}$, относительная погрешность $\approx 3.21 \times 10^{-14}$

<details>
<summary>💡 Почему обратное суммирование точнее?</summary>

<br>

При прямом суммировании накопленная сумма быстро растёт, и мелкие слагаемые в хвосте **поглощаются** мантиссой `float64`. При обратном — мелкие числа успевают сложиться, пока масштаб суммы ещё мал.

```
1e16 + 1 = 1e16   ← единица пропала (мантисса забита)
```

</details>

### 🔴 Лаб. 3 — Решение СЛАУ методом LU-разложения

> 📓 [`task_3.ipynb`](task_3.ipynb)

Решение системы $Ax = b$ методом LU-разложения на чистом Python (циклы + условные операторы).

**⚙️ Алгоритм (3 этапа):**

```
┌─────────────────────────────────────────────────┐
│  Этап 1 │  A = LU          (LU-разложение)      │
│  Этап 2 │  Ly = b  → y     (прямая подстановка) │
│  Этап 3 │  Ux = y  → x     (обратная подстановка)│
└─────────────────────────────────────────────────┘
```

**🔑 Ключевая формула:**

$$l_{jk} = \frac{a_{jk}}{a_{kk}}, \quad k < j \leq n$$

**✅ Проверки:**
- $L \cdot U = A$ — точное совпадение
- Невязка $\|Ax - b\|_\infty = 0$
- Сверка с `numpy.linalg.solve`

### 🟣 Лаб. 4 — Решение СЛАУ методом Якоби

> 📓 [`task_4.ipynb`](task_4.ipynb)

Итерационный метод решения $Ax = b$ методом простой итерации (метод Якоби).

**⚙️ Алгоритм:**

$$x_i^{(k)} = \frac{1}{a_{ii}} \left( b_i - \sum_{j \neq i} a_{ij}\, x_j^{(k-1)} \right), \quad i = 1, \dots, n$$

**🔑 Особенности:**
- Начальное приближение: $x^{(0)} = (0, 0, \dots, 0)$
- Критерий остановки: $\|x^{(k)} - x^{(k-1)}\|_\infty < \varepsilon$
- Условие сходимости: строгое диагональное преобладание $|a_{ii}| > \sum_{j \neq i} |a_{ij}|$

**✅ Тесты:**
- Пример из лекции (2×2): сходимость за 20 итераций
- Матрица 4×4 с диагональным преобладанием: сходимость за 13 итераций
- Невязка $\|Ax - b\|_\infty < 10^{-6}$
- Сверка с `numpy.linalg.solve`

### 🟠 Лаб. 5 — Интерполяция Лагранжа

> 📓 [`task_5.ipynb`](task_5.ipynb)

Интерполяция функции $\sin(x)$ на отрезке $[0, \pi]$ многочленами Лагранжа с 3 и 5 узлами.

**⚙️ Формула Лагранжа:**

$$L(x) = \sum_{i=0}^{n-1} y_i \cdot l_i(x), \quad l_i(x) = \prod_{j \neq i} \frac{x - x_j}{x_i - x_j}$$

**📊 Результаты:**
- Максимальная погрешность уменьшается с увеличением числа узлов
- Полином точно проходит через все узлы интерполяции
- Визуализация базисных полиномов и проверка их свойств

### 🟡 Лаб. 6 — Численное интегрирование

> 📓 [`task_6.ipynb`](task_6.ipynb)

Вычисление интеграла $\int_0^1 e^x dx$ методами центральных прямоугольников и трапеций.

**⚙️ Методы:**

| Метод | Формула | Порядок точности |
|:-----:|---------|:----------------:|
| Центр. прямоугольники | $h \sum f(x_i + h/2)$ | $O(h^2)$ |
| Трапеции | $\frac{h}{2}[f(a) + f(b) + 2\sum f(x_i)]$ | $O(h^2)$ |

**📈 Анализ:**
- Число отрезков: 4, 8, 16, 32, 64
- Погрешность уменьшается в 4 раза при удвоении числа отрезков
- Геометрическая визуализация методов

### 🔵 Лаб. 7 — Правило Рунге

> 📓 [`task_7.ipynb`](task_7.ipynb)

Проверка оценки погрешности по правилу Рунге для метода трапеций.

**⚙️ Правило Рунге:**

$$|I - I_h| \approx \frac{|I_h - I_{h/2}|}{2^p - 1}$$

где $p = 2$ для метода трапеций.

**✅ Проверка:**
- Пары разбиений: (4, 8), (8, 16), (16, 32)
- Сравнение оценки Рунге с фактической погрешностью
- Отношение оценки к фактической погрешности близко к 1

### 🟣 Лаб. 8 — Решение нелинейных уравнений

> 📓 [`task_8.ipynb`](task_8.ipynb)

Нахождение ширины функции $f(x) = x \cdot e^{-x}$ на полувысоте методом простой итерации.

**⚙️ Алгоритм:**
1. Найти максимум функции: $x_{\max} = 1$, $f_{\max} = e^{-1}$
2. Вычислить полувысоту: $f_{1/2} = f_{\max}/2$
3. Решить $f(x) = f_{1/2}$ для двух корней
4. Вычислить ширину: $\Delta x = x_2 - x_1$

**🔑 Метод простой итерации:**
- Преобразование к виду $x = \phi(x)$
- Итерации: $x_{n+1} = \phi(x_n)$
- Точность: $\varepsilon = 10^{-2}$

### 🟢 Лаб. 9 — Метод Зейделя

> 📓 [`task_9.ipynb`](task_9.ipynb)

Итерационный метод решения СЛАУ с использованием уже обновлённых значений текущей итерации.

**⚙️ Формула:**

$$x_i^{(k)} = \frac{1}{a_{ii}} \left( b_i - \sum_{j<i} a_{ij} x_j^{(k)} - \sum_{j>i} a_{ij} x_j^{(k-1)} \right)$$

**🔑 Отличие от Якоби:**
- Зейдель использует $x_j^{(k)}$ для $j < i$ (уже обновлённые)
- Якоби использует только $x_j^{(k-1)}$ (предыдущая итерация)
- Зейдель обычно сходится быстрее

**✅ Тесты:**
- Матрицы 2×2, 3×3, 4×4 с диагональным преобладанием
- Сравнение скорости сходимости с методом Якоби
- Ускорение в 1.5-2 раза

### 🟠 Лаб. 11 — Формула Симпсона

> 📓 [`task_11.ipynb`](task_11.ipynb)

Проверка свойства формулы Симпсона: точность для полиномов степени ≤ 3.

**⚙️ Формула Симпсона:**

$$\int_a^b f(x)dx \approx \frac{h}{3}\left[f(x_0) + 4\sum_{i=1,3,5}f(x_i) + 2\sum_{i=2,4,6}f(x_i) + f(x_n)\right]$$

**✅ Проверка:**
- Для $f(x) = x^3$: погрешность $\approx 0$ (машинная точность)
- Для $f(x) = e^x$: погрешность ненулевая, порядок $O(h^4)$
- Визуализация сходимости
